# Per-step ACTIVE-STATE size: Clifft vs TTN vs near-Clifford

Companion to `reports/per_step_memory_3way` (which plots resident **bytes**). This
folder plots, per runtime step, the **active-state size** — how big a dense state
each backend must hold — in two views:

* **`*_per_step_dim_linear.png`  ← the linear view.** y = the dense-equivalent
  **dimension** (number of complex amplitudes) on a **linear** axis.
* `*_per_step_qubits_linear.png` — the same thing as `log2(dimension)` in **qubits**
  (a log view of the dimension; handy when the linear gap is too large to read).

| line | dimension (linear PNG) | qubits (log PNG) | meaning |
|---|---|---|---|
| **Clifft** (crimson) | `2^k` | `k` active idents | dense `2^k` active state |
| **TTN** (steelblue) | `stored_bytes/16` | `log2(stored/16)` | whole tensor network |
| **near-Clifford** (seagreen) | `2^max_block` | `max_block` | genuine magic; **0/1** for a pure stabilizer circuit |

Generated by `ttn_backend/scripts/per_step_memory_compare.py --metric active_dim`
(and `--metric qubits`), linear y, one shot, carving_leaf layout. Table:
`SUMMARY_TABLE.md`.

## Reading the graphs

1. **Coherent-error families — near-Clifford collapses the active state.** The
   coherent rotations are *equatorial* (never create Z-basis magic), so the genuine
   magic register is far below the full active width: `d5_r1 → 0` (vs Clifft 13,
   TTN 11.5), `d5_r5 → 12` (vs Clifft 24, TTN 23.8). Clifft and TTN both pay for the
   full **stabilizer** entanglement of the syndrome state; the near-Clifford tableau
   absorbs that for free and only the non-Clifford core remains. The clifft line
   **oscillates per QEC round** (each round expands ancillas → `k`↑, measures them →
   `k`↓); on the *dimension* (linear) plot the inter-round dips to `k≈12` render as
   ~0 next to the `2^24` peaks, so the red curve looks like repeated pulses — that is
   the real round structure, not an artifact.

2. **All-magic families — block factoring + frame reduction give parity-to-win, no
   regression.** Even when every active ident is a genuine magic qubit, the magic
   register *factors*: `distillation` transient `2^3` (resident `2^2`) vs Clifft `2^5`
   is a **4×/8× win**; `cultivation_d3` transient `2^4` = Clifft `2^4` is **parity**
   (resident `2^3`, a 2× settled win); `cultivation_d5` transient `2^10` = Clifft
   `2^10` is **parity** (resident settles to `2^9`). The magic is irreducible at the
   `cultivation` limit, so **parity is the ceiling there** — frame reduction's job is
   to guarantee the transient peak never *exceeds* Clifft (it does not, on any
   circuit), not to manufacture a win where the state is genuinely all-magic. The
   one residual is a per-measurement, **sub-peak** transient `+1` on `cultivation_d5`
   (at the lone measurement where Clifft's local rank dips to 2) — it never reaches
   the global peak; see the measurement-dependency report.

3. **surface_d7_r7 is frame-only.** It compiles to **zero** active idents (pure
   Clifford + Pauli noise stays entirely in the Pauli frame), so all three backends
   carry ~no active state. The TTN line is dropped (its carving seeder recurses to
   the Python limit / builds an empty network on this degenerate input).

## Two honesty notes (read before trusting a curve)

**(a) TTN on coherent_d5_r5 is truncated at step ~2289 / 3228.** The full-circuit
TTN run does not complete: at the entanglement peak one carving step is a single
giant tensor contraction that does not finish in any reasonable budget (and the
per-step timeout only fires at step boundaries, so it cannot interrupt it). The TTN
line therefore **stops** at the last step it actually executed — it is *not*
forward-filled across the unexecuted tail (an earlier version did, which made TTN
look like it "stayed" at 217 MiB to the end). Clifft and near-Clifford run the full
3228 steps. Every other circuit's TTN completes (100%).

**(b) near-Clifford counts only the GENUINE ACTIVE magic — and frame reduction now
actively peels the dead residue.** A measured-out magic qubit used to linger in the
register: the `_ag_measure` path resolves the outcome via the stabilizer tableau and
left the qubit resident, and a *naive* drop corrupts the trajectory (register
membership is coupled to the tableau). Two things now handle it. First, the plotted
line is still computed **read-only**, excluding blocks all of whose qubits are already
measured out (a dead tensor factor), so it drops to ~0 at the end without perturbing
the run. Second — and this is the new part — **frame reduction (`decouple_demote`,
default ON) peels the residue exactly**: at each magic measurement it decouples the
demoted qubit (including the `q∉supp` parity-slaved case, via a GF(2)-found stabilizer
+ CNOTs), an exact identity insertion that is state- and distribution-exact (it changes
only RNG ordering, not the outcome distribution — `verify_block` fidelity 1.0). So the
residue no longer merely *accounted-around*; it is genuinely removed, which is why the
settled **resident** now drops a further qubit below the transient on every all-magic
circuit (`cultivation_d3` 4→3, `cultivation_d5` 10→9, distillation 3→2).
