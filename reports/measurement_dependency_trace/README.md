# Measurement-level non-Clifford dependency trace

**Message.** Each measurement depends on only a *small anticommutation-connected
core* of the deferred non-Clifford rotations, so the near-Clifford (NC) state size
stays far below Clifft's active dense rank at that same measurement.

This is a **measurement-level** view (one row = one measurement), deliberately *not*
a circuit-level aggregate bar chart. The key per-measurement quantities are
`nc_core_rot` and `nc_core_qubits`.

Generated by `nearclifford_backend/scripts/measurement_dependency_trace.py`
(block backend, 1 representative shot, `seed=12345`).

**Frame-reduction (`decouple_demote`) is ON by default** (state-exact, distribution-exact;
§11). It peels the just-demoted qubit at each magic measurement so dead residue does not
linger, which removes the per-measurement memory loss: `distillation` and `cultivation_d3`
now have `nc ≤ clifft` at **every** measurement (no tail crossover). `cultivation_d5`
(excluded here) is only partially fixed and still needs the full frame reduction.

## Files

- `measurement_dependency_trace_<circuit>.csv` — one row per measurement.
- `measurement_dependency_summary.csv` — one row per circuit.
- `measurement_dependency_<circuit>.png` — 2-panel plot (see below).

## Circuits

| tier | circuits | note |
|---|---|---|
| **headline** | `coherent_d5_r5` (most detailed), `coherent_d3_r3`, `distillation`, `cultivation_d3` | thesis holds: NC state < Clifft active dense (or parity for `cultivation_d3`) |

## Per-measurement columns (`..._trace_<circuit>.csv`)

| column | meaning |
|---|---|
| `circuit`, `shot_id`, `meas_idx` | circuit, shot, measurement index in execution order |
| `step_id`, `record_id`, `meas_qubit` | bytecode step, classical record index, measured NC qubit |
| `pending_rot_before` | # deferred non-Clifford rotations pending just before this measurement |
| `nc_core_rot` | # of those rotations in this measurement's anticommutation core (the ones it actually depends on) |
| `not_needed_this_meas` | `= pending_rot_before − nc_core_rot` (pending rotations THIS measurement does **not** touch) |
| `nc_core_qubits` | # qubits in the factored entangled **block** the core materialises (the actual dense cost of the core) |
| `nc_state_log2_transient` | log2 of the full NC magic state at this measurement = largest block at flush peak |
| `nc_state_log2_resident` | log2 of the settled NC state after measurement + purge |
| `clifft_active_rank_at_meas` | Clifft active rank `k` at this measurement (the dense `2^k` Clifft would hold) |
| `state_gap_log2` | `= clifft_active_rank_at_meas − nc_state_log2_transient`; **per-measurement memory gain = `2^state_gap_log2`** |
| `core_rot_ids`, `core_qubit_ids` | the actual rotation ids / qubit ids in the core (space-separated) |

## Summary columns (`..._summary.csv`)

`total_rotations`, `ever_flushed_rotations`, `never_flushed_rotations`,
`never_ratio`, `max_pending_rot_before`, `max_nc_core_rot`, `max_nc_core_qubits`,
`max_clifft_active_rank`, `max_nc_state_log2_transient`,
`max_nc_state_log2_resident`, `max_state_gap_log2`, `avg_state_gap_log2`,
`sum_2_nc_core_qubits`.

## The 2 plots (per circuit)

x-axis = `meas_idx` (measurements in execution order). No shading.

**(1) Rotation dependency** — `pending_rot_before` (grey) vs `nc_core_rot` (red): each
measurement depends on only a small subset of the deferred rotations.

**(2) State size needed at each measurement** — `clifft` (black) vs `nc` (red): the
dense state dimension (log2) each backend must materialise to do that measurement.
`nc` here is `nc_state_log2_transient` (the size actually needed at the measurement;
no flush/resident split). The gap = the memory advantage (`clifft / nc = 2^gap`).

(The full breakdown — resident, dependency-core block, deferred/free regions, and
`state_gap_log2` — remains in the trace CSV for anyone who wants it.)

## Do not conflate

- **`never_flushed`** is a whole-circuit property (rotations *no* measurement ever
  depends on) → summary only.
- **`not_needed_this_meas`** is per-measurement (`pending_before − core`).
- Clifft is compared on **active rank / state size**, never on rotation count.
